## Core decisions

The Suki module introduces a second user type ("customer") that is NOT a Supabase Auth user. Customers are rows in the `public.customers` table, authenticated via custom Edge Functions that issue app-level session tokens stored in `expo-secure-store`. Business owners remain standard Supabase Auth users.

**Why:** Customers are sub-accounts of a business owner, not independent tenants. Using Supabase Auth for them would bloat the `auth.users` table and complicate RLS — a custom session token pattern is cleaner and gives full control over the one-time QR enforcement rule.

**How to apply:** Never try to call `supabase.auth.signIn()` for a customer. All customer auth goes through Edge Functions: `generate-customer-qr`, `consume-customer-qr`, `authenticate-customer`. Customer session token is stored in `expo-secure-store` under key `suki-customer-session`.

## One-time QR token invariant

A token in `customer_qr_tokens` is permanently invalidated once `consumed_at IS NOT NULL`. This is enforced at the PostgreSQL level via the `consume_qr_token()` function using `SELECT ... FOR UPDATE` to prevent race-condition double-scans. The client-side expiry timer (15 min) is UX only — the server re-validates `expires_at` independently.

## Business code

Each business owner gets a unique 8-char uppercase alphanumeric code from the `business_codes` table. Customers enter this code during registration to link to the correct business. The code uses a reduced alphabet (no O/0, I/1) to avoid visual confusion. Generated by the `generate_business_code()` SQL function.

## Route groups

- `/(customer)/` — new Stack group for the customer-facing app (home, products, cart, orders, profile, verify-id, verify-liveness). Guarded: redirect to `/(auth)/login` if not customer-logged-in.

## Pay Later gating

`canUsePayLater(customer)` returns true only when `verification_status === 'VERIFIED'` AND `pay_later_enabled === true`. The server-side gate is in the `place-online-order` Edge Function — never trust the client check alone. When Pay Later order is placed, `createCreditSaleFromPOS()` from the existing credit repository is called to create a credit ledger entry.

## SQLite stock reduction for online orders

`reduceProductStockFromOnlineOrder()` in `online_order.repository.ts` uses movement_type `'OUT'` with reference_type `'ONLINE_ORDER'`. Called AFTER Supabase insert succeeds. On SQLite failure, failure is queued in `offline_sync_queue` with type `'STOCK_REDUCTION'`. `initializeInventory()` is called after success to refresh in-memory store.

## Library choices

- QR generation: `react-native-qrcode-svg` + `react-native-svg`
- Camera/QR/barcode scanning: `expo-camera` v15 (`CameraView` component, SDK 54)
- ID OCR: `@react-native-ml-kit/text-recognition` (on-device, free, offline, requires dev build)
- Face liveness: `expo-face-detector` + `expo-camera` front camera (client-side, yaw/pitch angles)
- Session storage: `expo-secure-store` (NOT AsyncStorage for tokens)
- Password hashing: bcrypt cost 12 in Deno Edge Functions (never client-side)
