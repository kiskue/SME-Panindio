# SME-Panindio Bug Hunter — Memory Index

## Project
- [Product Ingredients Persistence Pattern](./project_ingredient_link_pattern.md) — `replaceProductIngredients` must be called after `addItem` succeeds; two-step by design due to FK constraint.

## Patterns
- [Mark Paid on New Entry](./pattern_mark_paid_new_entry.md) — Use the id returned by `upsertLog`, not `editingLog.id`, when chaining markPaid; `editingLog` is null for new entries and the paid flag is silently skipped otherwise.
- [useState sentinel + fallback effect for late-hydrating store data](./pattern_usestate_late_hydration.md) — When a useState initialiser reads from a Zustand store that may still hydrating, use '' as a sentinel and add a fallback useEffect([types, visible]) to backfill once data arrives.
- [Store Initialization Pattern](./pattern_store_initialization.md) — Every new Zustand store with a bootstrap function must be added to `initializeStores()` in `src/store/index.ts` or it will not be warm-started before screens mount.
- [SQLite JOIN Safety](./pattern_sqlite_left_join.md) — All consumption/audit log queries that JOIN `inventory_items` must use LEFT JOIN; INNER JOIN silently drops rows when items are soft- or hard-deleted.
- [SQLite Migration vs Schema Registry Drift](./pattern_sqlite_migration_schema_drift.md) — Schema registry CREATE TABLE runs before migrations on every launch; ALTER TABLE ADD COLUMN in a migration must not duplicate a column already in the schema file or SQLite crashes with "duplicate column name".
- [Quantity Badge Sign Convention](./pattern_quantity_badge_sign.md) — `quantityConsumed` is negative for RETURN events; check `< 0` first to show `-N`; never use `+` prefix — original code had the ternary inverted.
- [CategoryNavCard count badge wiring](./pattern_category_nav_count_badge.md) — Consumption Logs card in `inventory/index.tsx` had `count={0}` hardcoded; all CategoryNavCard count props must come from store selectors, never literals.
- [Store Action Re-throw Convention](./pattern_store_action_rethrow.md) — Zustand actions awaited from UI try/catch blocks must re-throw after setting error state, or the form's catch block is dead code and always shows success.
- [Consumption Log and Stock Deduction Are Separate Operations](./pattern_consumption_stock_deduction.md) — `createConsumptionLog` only writes an audit row. Any trigger that changes stock must also call `adjustItemQuantity` then sync the inventory store cache.
- [useEffect Initial Prop via Ref Pattern — CORRECTED](./pattern_useeffect_initial_prop_ref.md) — The ref-sync + visibility-effect approach is BROKEN (ref-sync effect runs after the visibility effect in the same commit; ref is stale at read time). Use prevVisibleRef to detect the false→true transition and include the seed prop directly in the dep array.
- [Post-Save Side Effect Isolation](./pattern_post_save_side_effect_isolation.md) — `markPaid` and `loadMonthlySummary` after `upsertLog` must each have their own try/catch; a failure in either must not surface as "failed to save entry" to the user.
- [Zustand Action Proxy Object Anti-Pattern](./pattern_zustand_action_proxy_object.md) — Never group Zustand actions into a plain `const store = { ... }` object; the new object reference on every render defeats every useCallback that lists `store` as a dep. Extract actions as individual `const` variables.
- [Modal setSaving on Unmounted Component](./pattern_modal_setSaving_unmounted.md) — `finally { setSaving(false) }` in a bottom sheet must guard with a `mountedRef` because `onClose()` can unmount the component before `finally` executes.
- [utility_logs Period Filter Convention](./pattern_utility_logs_period_filter.md) — KPI aggregation on `utility_logs` must use `period_year`/`period_month` integers, never `paid_at`/`created_at` date ranges; timestamp columns are for audit/filter only.
- [Dashboard Double-Load Anti-Pattern](./pattern_dashboard_double_load.md) — `setPeriod` already calls `loadDashboard` internally; never add a `useEffect([period])` that also calls `loadDashboard` — it causes a double-fetch race on every period tap.
- [SQLite JOIN column alias drift](./pattern_join_column_alias_drift.md) — JOIN query column names must match the actual schema (e.g. `rm.is_synced` not `rm.synced_at`); alias names in SQL must match what the mapping code reads; all required domain type fields must be populated in inline object literals built from JOIN results.
- [Always import from @/types barrel](./pattern_type_barrel_import.md) — All files must use `from '@/types'`, never `from '@/types/raw_materials.types'` or any other direct subpath. Grep for `from '@/types/'` (with trailing slash) after any new feature to catch violations.
- [getDatabase Concurrent-Open Race](./pattern_getdatabase_concurrent_open.md) — `getDatabase()` must guard with an in-flight promise (`_opening`) in addition to the `_db !== null` guard; without it, two concurrent callers that both see `_db === null` before the first open resolves will each call `openDatabaseAsync`, producing two file handles and potential WAL conflicts.
- [Dashboard Not Loaded in initializeStores](./pattern_dashboard_not_in_initializestores.md) — `useDashboardStore.loadDashboard` must NOT be called inside `initializeStores`; the dashboard screen's own mount `useEffect` owns the first load. Adding it to `initializeStores` causes a double-fetch race on every cold start.
- [Expo SQLite Empty Params Array Causes prepareAsync Rejection](./pattern_expo_sqlite_empty_params.md) — Never pass `[]` as second arg to `getAllAsync`/`getFirstAsync` when the SQL has no `?` markers; use the single-argument overload instead. The two-argument path always fires `prepareAsync + bindAsync`, which the native module rejects on param-less statements.
- [Zustand Derived Selector New Reference Infinite Loop](./pattern_zustand_derived_selector_new_ref.md) — Never call a selector that runs `.filter()`/`.map()` as a naked `useStore(selector)` — each call returns a new array reference, Zustand's `useSyncExternalStore` always sees a change, and React loops infinitely. Subscribe to the stable source array + primitive filter values separately; apply `.filter()` inside `useMemo`. Alternatively wrap with `useShallow`.
- [Consumption Summary Reason Filter](./pattern_consumption_summary_reason_filter.md) — `getRawMaterialConsumptionSummary(reason?)` must accept an optional reason so per-material cost totals match the active filter; `getWasteRawMaterialCost()` is intentionally always global (unfiltered). `fetchSupportingData(filters)` passes `filters.reason` to summary but not to waste cost.
- [Cost Snapshot in Audit Log Rows](./pattern_cost_snapshot_in_log_rows.md) — Audit log tables must store `cost_per_unit` as a snapshot at INSERT time. Computing cost at query time via JOIN gives wrong totals when the master price is 0 or has changed. Use `SUM(cl.quantity_used * cl.cost_per_unit)` — never `SUM(cl.quantity_used * COALESCE(rm.cost_per_unit, 0))`.
- [Schema Registry Must Run After Migrations](./pattern_schema_registry_before_migrations_crash.md) — Running the schema registry before migrations crashes on existing installs when an index references a column added by a pending migration. `initDatabase.ts` order: create tracking table → run migrations → apply schema registry.
- [Bottom Sheet Layout Contract](./pattern_bottom_sheet_layout.md) — KAV must be `flex:1` with NO `justifyContent`; `Animated.View` must be `position:'absolute', bottom:0`; KAV `behavior` must be `undefined` on Android (not `'height'`). `justifyContent:'flex-end'` on KAV + `behavior='height'` on Android causes the sheet to jump off-screen when keyboard appears.
- [Gorhom Handle Chrome Occludes First Content Row](./pattern_gorhom_handle_chrome_occlusion.md) — `SafeAreaView edges={['bottom']}` does NOT compensate for the gorhom handle container (~24 pt). `SafeAreaView` must have `paddingTop: 24` when `showHandle=true`, else the sheet title is painted behind the drag indicator.
